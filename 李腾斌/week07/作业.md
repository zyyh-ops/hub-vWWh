# 作业1
## 问题1：BERT 文本分类与实体识别的关系

### 模型架构

本项目实现了一个 **联合模型（Joint Model）** 架构，在 `model.py` 中定义：

```python
class BertForIntentClassificationAndSlotFilling(nn.Module):
    def __init__(self, config):
        super(BertForIntentClassificationAndSlotFilling, self).__init__()
        self.bert = BertModel.from_pretrained(config.bert_dir)
        
        # 意图分类头（句子级别）
        self.sequence_classification = nn.Sequential(
            nn.Dropout(config.hidden_dropout_prob),
            nn.Linear(config.hidden_size, config.seq_num_labels),
        )
        
        # 槽位填充头（Token级别）
        self.token_classification = nn.Sequential(
            nn.Dropout(config.hidden_dropout_prob),
            nn.Linear(config.hidden_size, config.token_num_labels),
        )

    def forward(self, input_ids, attention_mask, token_type_ids):
        bert_output = self.bert(input_ids, attention_mask, token_type_ids)
        pooler_output = bert_output[1]  # [CLS] 的表示，768维
        token_output = bert_output[0]   # 所有token的表示，seq_len × 768
        
        seq_output = self.sequence_classification(pooler_output)
        token_output = self.token_classification(token_output)
        return seq_output, token_output
```

### 两者的关系对比

| 方面 | 文本分类（意图识别） | 实体识别（槽位填充） |
|------|---------------------|---------------------|
| **输入** | pooler_output（[CLS] 的 768 维向量） | token_output（每个 token 的 768 维向量） |
| **粒度** | 句子级别 | Token 级别 |
| **输出维度** | 24 个意图类别 | 117 个 BIO 标签（O + 58×2） |
| **共享部分** | BERT 编码器 | BERT 编码器 |
| **任务类型** | 多分类 | 序列标注 |

### 关系总结

1. **共享编码器**：两个任务共享同一个 BERT 编码器，通过联合训练相互促进
2. **不同输出头**：在 BERT 之上分别添加两个独立的分类头
3. **互补信息**：意图识别提供全局语义信息，槽位填充提供局部实体信息

### 分别使用的 Loss

在 `main.py` 中定义了损失函数：

```python
self.criterion = nn.CrossEntropyLoss()

# 训练时计算损失
seq_loss = self.criterion(seq_output, seq_label_ids)      # 意图识别损失
token_loss = self.criterion(active_logits, active_labels)  # 槽位填充损失
loss = seq_loss + token_loss                               # 总损失
```

**两个任务都使用交叉熵损失（CrossEntropyLoss）：**

- **意图识别 Loss**：
  - 输入：`(batch_size, 24)` 的 logits
  - 标签：`(batch_size,)` 的意图 id
  - 本质：24 类的多分类问题

- **槽位填充 Loss**：
  - 通过 `attention_mask` 过滤掉 padding 部分（只计算有效 token）
  - 输入：有效 token 的 logits
  - 标签：有效 token 的 BIO 标签 id
  - 本质：117 类的 token 级多分类问题

## 问题2：多任务训练 Loss 相加的问题及解决方案

### 简单相加的坏处

#### 1. Loss 量级不平衡

- `seq_loss`：每个样本产生 **1 个** 损失值
- `token_loss`：每个样本产生 **多个 token** 的损失值累加
- 结果：token_loss 通常比 seq_loss 大很多，主导训练方向

#### 2. 任务难度不同

- 意图识别：24 分类问题，相对简单
- 槽位填充：117 分类 + 序列依赖，更加复杂
- 结果：简单任务可能过早收敛，复杂任务欠拟合

#### 3. 收敛速度不一致

- 两个任务的学习曲线可能完全不同
- 一个任务收敛后继续训练可能导致过拟合
- 另一个任务可能还需要更多训练

#### 4. 梯度冲突

- 两个任务对共享参数的梯度方向可能相反
- 导致优化过程中的"拉锯战"
- 整体优化效率降低，可能出现负迁移

### 解决方案

#### 方案一：固定权重加权求和（最简单）

```python
# 手动设置权重
alpha = 1.0  # 意图识别权重
beta = 0.5   # 槽位填充权重

loss = alpha * seq_loss + beta * token_loss
```

**优点**：实现简单  
**缺点**：需要手动调参，不够灵活

#### 方案二：不确定性加权（Uncertainty Weighting）

基于论文《Multi-Task Learning Using Uncertainty to Weigh Losses for Scene Geometry and Semantics》

```python
class UncertaintyWeightedLoss(nn.Module):
    def __init__(self):
        super().__init__()
        # 可学习的 log(σ²) 参数
        self.log_var_seq = nn.Parameter(torch.zeros(1))
        self.log_var_token = nn.Parameter(torch.zeros(1))
    
    def forward(self, seq_loss, token_loss):
        # 精度 = 1/σ²
        precision_seq = torch.exp(-self.log_var_seq)
        precision_token = torch.exp(-self.log_var_token)
        
        # 加权损失 + 正则项
        loss = precision_seq * seq_loss + self.log_var_seq + \
               precision_token * token_loss + self.log_var_token
        return loss
```

**优点**：自动学习权重，理论基础扎实  
**缺点**：增加可学习参数

#### 方案三：梯度归一化（GradNorm）

基于论文《GradNorm: Gradient Normalization for Adaptive Loss Balancing in Deep Multitask Networks》

```python
class GradNormLoss:
    def __init__(self, num_tasks=2, alpha=1.5):
        self.num_tasks = num_tasks
        self.alpha = alpha
        self.weights = nn.Parameter(torch.ones(num_tasks))
        self.initial_losses = None
    
    def compute_grad_norm(self, loss, shared_params):
        """计算梯度范数"""
        grads = torch.autograd.grad(loss, shared_params, retain_graph=True)
        grad_norm = torch.norm(torch.stack([g.norm() for g in grads]))
        return grad_norm
    
    def update_weights(self, losses, shared_params):
        # 记录初始损失
        if self.initial_losses is None:
            self.initial_losses = [l.detach() for l in losses]
        
        # 计算相对训练进度
        loss_ratios = [l / l0 for l, l0 in zip(losses, self.initial_losses)]
        avg_ratio = sum(loss_ratios) / len(loss_ratios)
        inverse_rates = [r / avg_ratio for r in loss_ratios]
        
        # 计算梯度范数
        grad_norms = [self.compute_grad_norm(l * w, shared_params) 
                      for l, w in zip(losses, self.weights)]
        avg_grad_norm = sum(grad_norms) / len(grad_norms)
        
        # 更新权重
        for i in range(self.num_tasks):
            target_norm = avg_grad_norm * (inverse_rates[i] ** self.alpha)
            self.weights.data[i] *= (target_norm / grad_norms[i]).detach()
        
        # 归一化权重
        self.weights.data = self.weights.data / self.weights.data.sum() * self.num_tasks
```

**优点**：动态平衡梯度，自适应性强  
**缺点**：计算开销大，实现复杂

#### 方案四：交替训练

```python
def train_alternating(model, train_loader, epochs):
    for epoch in range(epochs):
        for batch in train_loader:
            seq_output, token_output = model(batch)
            seq_loss = criterion(seq_output, batch['seq_labels'])
            token_loss = criterion(token_output, batch['token_labels'])
            
            if epoch % 2 == 0:
                # 偶数轮只训练意图识别
                seq_loss.backward()
            else:
                # 奇数轮只训练槽位填充
                token_loss.backward()
            
            optimizer.step()
            optimizer.zero_grad()
```

**优点**：避免梯度冲突  
**缺点**：训练效率低，无法充分利用任务间的协同

#### 方案五：PCGrad（投影冲突梯度）

基于论文《Gradient Surgery for Multi-Task Learning》

```python
def pcgrad_backward(losses, shared_params):
    grads = []
    for loss in losses:
        grad = torch.autograd.grad(loss, shared_params, retain_graph=True)
        grads.append(torch.cat([g.view(-1) for g in grad]))
    
    # 检测并处理梯度冲突
    for i in range(len(grads)):
        for j in range(len(grads)):
            if i != j:
                dot = torch.dot(grads[i], grads[j])
                if dot < 0:  # 梯度冲突
                    # 投影到正交方向
                    grads[i] = grads[i] - (dot / (grads[j].norm() ** 2)) * grads[j]
    
    # 合并梯度
    final_grad = sum(grads)
    return final_grad
```

**优点**：理论上解决梯度冲突问题  
**缺点**：计算开销大，实现复杂

### 方案对比总结

| 方案 | 实现难度 | 计算开销 | 效果 | 推荐场景 |
|------|---------|---------|------|---------|
| 固定权重 | ⭐ | 低 | 一般 | 快速实验 |
| 不确定性加权 | ⭐⭐ | 低 | 好 | 生产环境推荐 |
| GradNorm | ⭐⭐⭐⭐ | 高 | 很好 | 研究场景 |
| 交替训练 | ⭐⭐ | 中 | 一般 | 任务差异极大时 |
| PCGrad | ⭐⭐⭐⭐ | 高 | 很好 | 任务冲突严重时 |

# 作业2 信息解析智能对话系统提示词设计

### 数据集分析

根据项目数据集，系统需要处理：

- **30 个领域**：music, app, weather, bus, train, flight 等
- **24 个意图**：LAUNCH, QUERY, PLAY, SEARCH, SEND 等
- **58 个槽位类型**：name, content, Src, Dest, artist, song 等

### 完整提示词

```
你是一个智能语音助手的语义理解模块，负责对用户的语音指令进行信息解析。
请对输入的文本完成以下三项任务：领域识别、意图识别、槽位填充。

## 任务说明

1. **领域识别（Domain）**：判断用户指令属于哪个应用领域
2. **意图识别（Intent）**：判断用户想要执行的操作类型
3. **槽位填充（Slot Filling）**：从文本中提取关键实体信息

## 可选标签
