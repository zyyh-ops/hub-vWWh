在`main.py`中,Trainer类下,Train函数中有这样一段代码: 

`loss = seq_loss + token_loss`

这里直接将意图识别的loss和实体识别的loss相加到了一块

### 分析缺点

1. 一个自然的想法是权重不平衡 ->
  + 意图是句子级，一个 batch 只有 batch_size 个标签；
  + 槽位(实体识别)是 token 级，一个 batch 有 batch_size * 有效长度 个标签，loss 的数值和梯度通常会更大。

结果：token_loss 很容易主导更新，模型更偏向优化实体识别，牺牲意图分类，反之也可能出现（看具体数据和实现）。

2. 训练过程中收敛速度不同 ->
  + 意图识别的任务往往很快 , loss短期且较小
  + 实体识别loss长期较大

结果: token_loss 很容易主导更新，模型更偏向优化实体识别，牺牲意图分类

### 解决方案

1. 其一就是添加固定的权重 ->
  `loss = alpha * seq_loss + beta * token_loss`

不过这样显得有些呆板,并且要经常根据结果修改两个权重值

3. 动态加权 -> 基于梯度大小的加权（如 GradNorm 等）

即监控各任务梯度的范数，如果某个任务梯度过大，就降低其权重，保持不同任务的训练速度相近。
