本次作业尝试把 04-government-advanced-rag 项目完整跑在一台 Linux GPU 云服务器上，包含：

+ vLLM（本地大模型服务，OpenAI 兼容）
+ Elasticsearch（文档与向量检索）
+ FastAPI 项目服务（main.py）


> vLLM : vLLM 是一个快速、易于使用的 LLM 推理和服务库。官方定位就是 fast and easy-to-use .
> 也可以这样说,vLLM 是“专门把大模型部署得更快、更省、更适合多人同时调用”的引擎层工具。它特别适合做聊天机器人、API 服务、批量推理、RAG 后端、以及需要较高并发的模型服务。

### 配置环境

1. 使用scp一次性传输整个项目目录

<img width="1229" height="538" alt="image" src="https://github.com/user-attachments/assets/636879c5-761c-4b9a-ba9c-034952cea6de" />
需要注意的是,这里终端提示输入密码后,scp在处理密码时是静默模式,即看不到输入,实际上是在接收密码的,所以输入后直接回车就可以了

或者鼠标右键粘贴

2. 在远程终端新建虚拟环境,与本地类似,均使用conda创建
采用如下命令 :
```bash
conda init bash
# source 读取并在当前环境下执行
# ~ 表示当前文件根目录
source ~/.bashrc
conda create -n rag python=3.10 -y
conda activate rag
pip install -U pip
# 可以提前写入txt文档,pip可以一键安装
pip install -r /root/autodl-tmp/04-government-advanced-rag/requirements.txt
# 安装vllm等所需库
pip install vllm sentence-transformers
```
而后还需安装`huggingface-hub`,视Transformer版本而定,此处pip代码类似

3. 下载model

将hf连接到国内镜像 
```bash
export HF_ENDPOINT=https://hf-mirror.com
```

下载embedding
```bash
huggingface-cli download BAAI/bge-small-zh-v1.5 \
  --local-dir /root/autodl-tmp/models/BAAI/bge-small-zh-v1.5
```

下载LLM
```bash
huggingface-cli download Qwen/Qwen2.5-7B-Instruct \
  --local-dir /root/autodl-tmp/models/Qwen/Qwen2.5-7B-Instruct
```

这一步需要等待较长的时间

根据下载位置调整config
```bash
cd /root/autodl-tmp/04-government-advanced-rag

# 1) 改 embedding 模型路径
# sed：Linux 里常用的文本处理工具，能按规则替换文件内容
# -i：直接修改原文件，不只是输出结果
# s|旧内容|新内容|：替换命令 s为substitute
sed -i 's|local_url: "/root/models/BAAI/bge-small-zh-v1.5/"|local_url: "/root/autodl-tmp/models/BAAI/bge-small-zh-v1.5"|' config.yaml

# 2) 改 LLM 模型名（和 vLLM --served-model-name 一致）
sed -i 's|llm_model: "models/Qwen/Qwen3.5-0.8B/"|llm_model: "qwen2.5-7b-instruct"|' config.yaml

# 3) 改 API key（和 vLLM --api-key 一致）
sed -i 's|llm_api_key: "43aa6570992d01fab078e157e07b4f52.ocKHwNGxhIBrDZD4"|llm_api_key: "rag-local-key"|' config.yaml
```
