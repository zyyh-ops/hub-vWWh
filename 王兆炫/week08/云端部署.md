#### 配置安装ES

因为直接安装过于缓慢,所以这里使用了aria2加速

```bash
# apt install 安装软件包
    # -y 自动回答yes
apt install -y aria2

# 使用aria2加速下载
    # aria2c：aria2 的命令行下载器
    # -x 16：最多开 16 个连接
    # -s 16：把文件拆成 16 个分片同时下载
    # -k 1M：每个分片大小设为 1MB
    # -c：断点续传，之前没下完可以接着下
    # \ 的作用是：换行继续写同一条命令，不表示命令结束。
cd /root/autodl-tmp
aria2c -x 16 -s 16 -k 1M -c \
  https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-8.15.0-linux-x86_64.tar.gz

# 解压
    # tar：打包/解包工具
    # -x：解包（extract）
    # -z：处理 gzip 压缩
    # -f：后面跟文件名
cd /root/autodl-tmp
tar -xzf elasticsearch-8.15.0-linux-x86_64.tar.gz
```

##### 踩坑记录

部署过程中遇到了两个问题：

**问题1：ES 不允许 root 用户直接运行**
- 需要创建一个普通用户 `esuser` 来启动 ES。

**问题2：`esuser` 无法访问 `/root` 目录**
- `/root` 目录权限是 `drwx------`（700），只有 root 能进入。
- 虽然 ES 文件本身有执行权限，但 `esuser` 根本穿透不了 `/root` 这一层目录，表现为 `Permission denied`。
- 解决方案：把 ES 目录移到 `/home/esuser/` 下，让 `esuser` 有完整访问权限。

**问题3：`elasticsearch.yml` 配置字段重复**
- 多次执行 `echo >> config/elasticsearch.yml` 追加配置，导致同一字段（如 `xpack.security.enabled`）出现多次。
- ES 启动时报 `Duplicate field 'xpack.security.enabled'`，直接崩溃。
- 解决方案：先用 `sed -i` 删除所有重复行，再用 `cat >>` 一次性追加一份干净配置。

##### 正确的启动流程

```bash
# 1) 创建用户
    # useradd：新增用户
    # -m：同时创建这个用户的家目录（/home/esuser）
    # id esuser || useradd -m esuser：
    #   先检查用户是否已存在，不存在才创建，避免重复报错
id esuser || useradd -m esuser

# 2) 把 ES 从 /root 移到 /home/esuser（esuser 能访问的地方）
    # mv：移动文件/目录
mkdir -p /home/esuser
mv /root/autodl-tmp/elasticsearch-8.15.0 /home/esuser/

# 3) 把整个 ES 目录的所有权改给 esuser
    # chown：修改文件/目录所属用户
    # -R：递归处理目录下所有文件和子目录
    # esuser:esuser：前面是用户，后面是组
chown -R esuser:esuser /home/esuser/elasticsearch-8.15.0

# 4) 写入干净的配置（先删重复项，再统一追加）
    # sed -i '/pattern/d'：删除文件中匹配 pattern 的所有行
    # cat >> file <<'EOF' ... EOF：把多行内容一次性追加到文件末尾
ES_YML=/home/esuser/elasticsearch-8.15.0/config/elasticsearch.yml

sed -i '/^xpack.security.enabled:/d' $ES_YML
sed -i '/^xpack.security.enrollment.enabled:/d' $ES_YML
sed -i '/^xpack.security.http.ssl.enabled:/d' $ES_YML
sed -i '/^discovery.type:/d' $ES_YML
sed -i '/^network.host:/d' $ES_YML
sed -i '/^http.port:/d' $ES_YML

cat >> $ES_YML <<'EOF'
xpack.security.enabled: false
xpack.security.enrollment.enabled: false
xpack.security.http.ssl.enabled: false
discovery.type: single-node
network.host: 0.0.0.0
http.port: 9200
EOF

# 5) 用 esuser 后台启动 ES
    # su - esuser -c "..."：切换成 esuser 用户，并执行引号内的命令
    # ES_JAVA_OPTS='-Xms2g -Xmx2g'：限制 JVM 堆内存为 2GB
    # -d：后台运行（daemon 模式）
su - esuser -c "ES_JAVA_OPTS='-Xms2g -Xmx2g' /home/esuser/elasticsearch-8.15.0/bin/elasticsearch -d"
```

等待约 30 秒后验证：
```bash
curl http://127.0.0.1:9200
# 看到含 "You Know, for Search" 的 JSON 返回就说明启动成功
```

#### 配置安装vLLM

在IDE中新开设一个终端,然后执行如下命令

```bash
conda activate rag

# vLLM相关配置
vllm serve /root/autodl-tmp/models/Qwen/Qwen2.5-7B-Instruct \
  --served-model-name qwen2.5-7b-instruct \
  --host 0.0.0.0 \
  --port 8000 \
  --api-key rag-local-key \
  --gpu-memory-utilization 0.9 \
  --max-model-len 4096
```
