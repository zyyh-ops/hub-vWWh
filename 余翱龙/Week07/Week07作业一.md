### **BERT 文本分类 vs 实体识别（NER）** 

一、两者到底是什么关系？ 

它们都是 **BERT 做自然语言理解（NLU）的经典任务**，结构高度相似，只有**输出层不一样**。

##### 共同点

- 都用 **BERT 编码器** 提取上下文特征
- 输入都是一段文本 → 得到每个 token 的 embedding
- 都属于 **监督学习任务**

##### 核心区别

##### ① 文本分类（Text Classification）

**给一整段话贴一个标签**

- 情感：正面 / 负面
- 意图：查询 / 投诉 / 预约
- 主题：体育 / 科技 / 娱乐

##### **只关心整句话的意思，不关心字 / 词的位置。**

##### ② 实体识别（Named Entity Recognition, NER）

**给每个字 / 词贴标签**，找出里面的实体

- 人名、地名、机构名、时间、金额等
- 常用标注：BIO / BIOES

**关心每个位置是什么实体，是序列标注任务。**

**分类 = 整句话一个标签**

**NER = 每个字一个标签**

CrossEntropyLoss 交叉熵损失

##### 实体识别 Loss

##### **CrossEntropyLoss **（代码实现中的的是这个）

但是这个更准确

##### 👉 **CRF + 对数似然损失（CRF Loss）**

为什么不用普通交叉熵？

- NER 是**强时序依赖**

  比如：B-PER 后面不能跟 B-LOC，必须是 I-PER

- CrossEntropy 独立预测每个 token，**不懂标签之间的规则**

- CRF 会学习标签之间的转移概率，保证输出合法序列

------

#### 多任务训练  loss = seq_loss + token_loss 有什么坏处，如何应对？

##### 量级失衡：两个 loss 不在一个 “数量级” 上

**seq_loss（意图分类）**：每句话只计算 1 次损失，数值通常在 `0.1~5` 之间（比如 CrossEntropyLoss，单样本损失）；

**token_loss（NER）**：每句话要计算所有 token 的损失（比如一句话有 20 个 token，CRF Loss/CrossEntropyLoss 会累加 / 平均这些 token 的损失），数值通常在 `5~20` 之间（甚至更高）。

**后果**：训练时模型会 “优先迎合” 数值更大的 NER loss，意图分类的 loss 被 “淹没”，导致意图识别任务学不到有效信息，甚至比单任务训练效果还差。

##### 应对方案 ：Loss 加权（最基础、最有效）

给两个 loss 加**可调节的权重系数**，平衡量级和重要性，公式改为：

total_loss = α * seq_loss + β * token_loss

##### 应对方案 ：Loss 归一化（消除量级差异）

对两个 loss 做 “标准化”，让它们的均值 / 方差统一到同一范围，常用两种方式：

##### 任务难度 / 收敛速度不同步

意图分类通常是 “简单任务”：BERT 的`[CLS]`向量已经能很好捕捉句子整体语义，收敛快；

NER 是 “复杂任务”：需要捕捉 token 间的依赖、实体边界，收敛慢（尤其是 CRF 层）。

**后果**：简单相加会导致 “快任务” 先收敛，“慢任务” 还在震荡，最终两个任务的效果都达不到最优。

##### 应对方案：训练策略优化 —— 分阶段训练（适配不同收敛速度）

##### 具体步骤：

1. **阶段 1（冻结 + BERT 适配）**：

   - 冻结 BERT 的所有参数，只训练两个输出层（意图线性层、NER 线性层 + CRF）；
   - 训练 1~2 个 epoch，让输出层先适配任务，避免 BERT 参数被大 loss 冲乱。

   

2. **阶段 2（微调 + 平衡 loss）**：

   - 解冻 BERT 的后 4~6 层（或全部层），用小学习率（2e-5~5e-5）微调；
   - 此时使用加权后的 loss，重点监控两个任务的指标，动态调整权重。

   

3. **阶段 3（精调 + 侧重）**：

   - 如果某个任务效果差，适当增大其 loss 权重，精调 1 个 epoch。

##### 此外：数据层面 —— 样本平衡

- 对意图分类的少数类样本做过采样，或对 NER 的稀有实体样本做增强；
- 保证每个 batch 中，不同意图、不同实体类型的样本分布尽量均衡；
- 比如：如果某 batch 中 “投诉” 意图样本少，手动补充几个，避免模型只学 “查询” 意图。
